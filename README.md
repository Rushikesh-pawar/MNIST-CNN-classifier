# MNIST CNN Classifier

A convolutional neural network that classifies handwritten digits (0-9) with **98.65% test accuracy** on the MNIST benchmark, paired with an interactive Streamlit demo where you can draw a digit and watch the model predict it in real time.

## Architecture

```
Input (28x28x1 grayscale)
  -> Conv2D(32, 3x3, ReLU)        ->  26x26x32
  -> Conv2D(64, 3x3, ReLU)        ->  24x24x64
  -> MaxPool(2x2)                 ->  12x12x64
  -> Dropout(0.25)
  -> Flatten                      ->  9216
  -> Dense(128, ReLU)
  -> Dropout(0.50)
  -> Dense(10, Softmax)           ->  class probabilities
```

**Optimizer:** SGD (lr = 0.01) &nbsp;|&nbsp; **Loss:** categorical cross-entropy &nbsp;|&nbsp; **Batch size:** 32 &nbsp;|&nbsp; **Epochs:** 10

The two stacked convolutions extract increasingly abstract digit features, max-pooling shrinks the spatial dimensions for efficiency, and the two dropout layers (one after the conv block, one in the dense head) keep training and validation accuracy from diverging.

## Quick Start

### 1. Clone and install

```bash
git clone https://github.com/<your-username>/MNIST-CNN-classifier.git
cd MNIST-CNN-classifier
python -m venv .venv && source .venv/bin/activate   # optional
pip install -r requirements.txt
```

### 2. Train the model

If `mnist_cnn_model.h5` is not already in the repo:

```bash
python train_model.py
```

This downloads MNIST through Keras, trains for 10 epochs, prints the test metrics, and saves the model to `mnist_cnn_model.h5`.

### 3. Launch the demo

```bash
streamlit run app.py
```

Open the URL Streamlit prints (usually [http://localhost:8501](http://localhost:8501)) and draw a digit.

## What I Learned

- Designing a small CNN from first principles: how each layer (convolution, pooling, dropout, dense) contributes to feature extraction and overfitting control.
- The importance of consistent preprocessing &mdash; the Streamlit demo had to replicate the exact training-time normalization (0-255 to 0-1) and reshape (28x28x1) for predictions to be meaningful.
- Bridging the gap between a Jupyter prototype and a deployable artifact: serializing the model, decoupling training from inference, and serving via a UI.
